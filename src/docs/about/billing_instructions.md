### 模型计费 

模型计费分以下几种

-   按次计费
-   按量计费
-   按秒计费
-   按百万像素计费
-   按图像计费
-   缓存/音频/推理等额外计费

::: warning ⚠️ 注意
本系统按照积分计算，所有金额均为积分换算而来，1 积分=$0.000002，最低消费为 1 积分，下面说明忽略了积分的转换。
:::

#### 按次计费 

按次计费是指每请求一次， 计费一次。

计费公式： 模型输入价格 \* 渠道折扣 \* 用户折扣 。

#### 按量计费 

按量计费是指根据请求使用的`tokens`数量计费。

计费公式：

费用输入输入价格输出输出价格费用\=(输入tokens1000×输入价格)+(输出tokens1000×输出价格)最终扣费费用渠道折扣用户折扣最终扣费\=费用×渠道折扣×用户折扣

#### 按秒计费 

按秒计费一般出现在视频生成模型中。 计费公式：

费用输入价格秒数费用\=输入价格×秒数最终扣费费用渠道折扣用户折扣最终扣费\=费用×渠道折扣×用户折扣

#### 按百万像素计费 

按百万像素计费一般出现在图像生成模型中。 计费公式：

费用宽度高度输入价格费用\=(宽度×高度1000000)×输入价格最终扣费费用渠道折扣用户折扣最终扣费\=费用×渠道折扣×用户折扣

#### 按图像计费 

按图像计费一般出现在图像生成模型中。他是按照生成的图像数量计费。 计费公式：

费用图像数量输入价格费用\=图像数量×输入价格最终扣费费用渠道折扣用户折扣最终扣费\=费用×渠道折扣×用户折扣

#### 其他计费 

##### 缓存/音频/推理等额外计费 

如果模型支持缓存/音频/推理等， 那么这些请求也可能会被计费。 系统的计费方法为：按照输入/输出价格的倍数计费。 计费公式：

输入输出输入输出（缓存音频推理倍数）输入/输出tokens\=输入/输出tokens+（(缓存/音频/推理tokens)×(倍数−1)）

例如：

-   输入 tokens 10000, 输出 tokens 20000, 缓存令牌 1000, 推理令牌 2000
-   缓存令牌倍率为 0.5， 推理令牌倍率为 1.5

那么实际输入 tokens 为：

实际输入实际输入tokens\=10000+(1000×(0.5−1))\=10000−500\=9500

那么实际输出 tokens 为：

实际输出实际输出tokens\=20000+(2000×(1.5−1))\=20000+1000\=21000

最后按照实际输入/输出 tokens 进行计费，计费方式见[模型计费](#模型计费)。

##### 阶梯计费 

根据输入tokens的值处于某个阶梯区间内，则按照该阶梯的价格进行计费。 一般存在与Gmini Pro /豆包 系列模型中。

例如doubao-seed-1-6-lite-251015：

-   输入tokens <= 32k时，输入价格是$0.0361645/M，输出是$0.289315/M
-   输入tokens > 32k && 输入tokens <= 128k时，输入价格是$0.0723288/M，输出是$0.482192/M
-   输入tokens > 128k时，输入价格是$0.144658/M，输出是$1.44658/M

计算公式和按量计费公式是一样的，只是根据输入tokens替换了输入/输出的价格

##### 字段倍率 

有些模型会根据请求字段的值来调整价格，一般存在于视频模型中。

例如kling-v1\_videos：

duration字段允许的值为 5/10秒，默认5秒，当选择 10秒时倍率为2。 mode字段允许的值为 std/pro，默认std，当选择pro时倍率为3.5。

如果用户请求了 duration=10, mode=pro，那么最终的价格倍率为 2 \* 3.5 = 7。

最终费用计算公式为：

费用基础费用字段倍率费用\=基础费用×字段倍率

再按照[模型计费](#模型计费)中的类型进行最终扣费计算。

### 举例 

假设：

-   您累计充值 1000 美元， 达到权益等级 Tier 3，权益折扣为 0.9。
-   OpenAI 官方渠道折扣为 0.88。
-   gpt-4o 是按量计费，输入价格为 0.0025 美元/k tokens， 输出价格为 0.01 美元/k tokens。
-   gpt-4o 的缓存令牌倍率为 0.5。

那么您的最终折扣为： 0.9 \* 0.88 = 0.792。

您请求一次 gpt-4o， 输入 1000 tokens， 输出 1000 tokens， 缓存令牌 500 tokens。

首先我们会计算您实际的输入/输出 tokens：

-   实际输入 tokens = 1000 + (500 \* (0.5 - 1)) = 1000 - 250 = 750
-   实际输出 tokens = 1000

计费：

费用费用\=(7501000×0.0025)+(10001000×0.01)\=0.001875+0.01\=0.011875

最终扣费：

0.011875×0.792\=$0.009405
